# README Product Sections Plan

## Scope

Add exactly three recruiter-friendly product-facing sections to `README.md`:

1. `Product Snapshot`
2. `What the Product Looks Like`
3. `60-Second Demo Flow`

This is a documentation-only product-legibility pass. It must preserve the current repository
truth:

- verifier-driven, durable, approval-gated incident-response runtime
- narrow `On-Call Copilot` product prototype
- panel-first product surface with a session-scoped secondary assistant
- one bounded `deployment-regression` loop
- no mature ops-platform claim
- no generic coding-agent or broad autonomous-remediation framing

## Planned changes

1. Insert `Product Snapshot` near the top of the README
   - summarize what On-Call Copilot is
   - define the core product value in plain language
   - clarify who it is for
   - keep the distinction from coding copilots explicit

2. Insert `What the Product Looks Like`
   - describe the product as a three-part decision surface:
     sessions list, incident detail/timeline/actions, and secondary assistant pane
   - include a clearly labeled screenshot/GIF placeholder because no committed asset is currently
     referenced from the README

3. Insert `60-Second Demo Flow`
   - provide a 5-7 step product demo sequence centered on console/session selection, blocked/ready
     clarity, assistant explanation, approval, verification, and handoff
   - keep it honest about the current narrow bounded loop

4. Consistency pass
   - verify wording matches `PRODUCT_BRIEF.md`, `POSITIONING.md`, `ONE_PAGER.md`,
     `PERSONAS.md`, `USER_JOURNEYS.md`, and `METRICS.md`
   - tighten any language that sounds broader, more autonomous, or more mature than the repo

## Non-goals

- no runtime changes
- no command-surface changes
- no new screenshots or visual assets
- no README rewrite into a marketing page
